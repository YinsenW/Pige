# Code Of Conduct

Pige is an open-source project for a local-first Personal Knowledge Agent. The project should be welcoming, careful with user data, and serious about design quality.

## Expected Behavior

Contributors, maintainers, users, and AI agents should:

- Be respectful, specific, and constructive.
- Assume good intent while still asking for evidence.
- Keep discussion focused on the product, code, design documents, tests, and user outcomes.
- Use synthetic or redacted examples when discussing bugs.
- Protect private vaults, source files, prompts, model responses, diagnostics, support bundles, API keys, and personal data.
- Credit prior work and upstream projects when relevant.
- Accept maintainer decisions about scope, safety, simplicity, and release timing.

## Unacceptable Behavior

The following is not acceptable:

- Harassment, personal attacks, threats, or discriminatory language.
- Publishing private user data, secrets, exploit details, or unredacted support material.
- Pressuring users to share private vaults, files, logs, prompts, or model responses.
- Deliberately bypassing security, privacy, permission, or moderation processes.
- Repeatedly derailing technical discussions after maintainers ask to refocus.
- Misrepresenting AI-generated work as reviewed human work when review is required.

## AI Agent Contributions

AI-generated contributions are welcome when they follow the same standards as human contributions.

AI agents must:

- Follow `AGENTS.md` and the task-specific design documents.
- Avoid fabricating test results, source links, upstream behavior, or maintainer decisions.
- Avoid exposing private user material in summaries, commits, issue comments, or handoff notes.
- Make uncertainty visible when a change depends on unverified assumptions.

## Reporting

For conduct concerns, contact the maintainers through the project's public or private maintainer channel once it is available.

For security vulnerabilities, do not use conduct channels or public issues. Follow [Security Policy](SECURITY.md).

For privacy or support concerns, see [Privacy And Data Use Policy](PRIVACY.md) and [Support Policy](SUPPORT.md).

## Enforcement

Maintainers may remove comments, close issues, edit titles, lock discussions, block accounts, or decline contributions that violate this code of conduct or put user data at risk.

Enforcement should be proportional, documented when appropriate, and focused on protecting the community and users.
