本地 RAG 设计

RAG 本地优先 Agent 基础能力

Pige 的知识取回不应该只表现为一个聊天回答。更合理的形态是增强搜索：先找到相关笔记和来源，再基于这些材料生成可追溯摘要。

v0.1 决策

  • 词法与元数据检索必须在未下载模型时可用。
  • 语义检索使用本地嵌入模型，模型由用户显式下载。
  • RAG 结果服务于首页对话、Note Agent 和整理工作流。
默认体验应该是用户只配置在线语言模型 API；嵌入和检索能力由 Pige 本地维护。

权限模式

安全 Agent YOLO

Pige 不引入“本会话允许”的心智，因为产品弱化了会话概念。更清楚的权限模型是：拒绝、仅本次允许、永久允许，并把永久允许限定在明确的资源范围内。

永久允许不应该是无限授权，它必须绑定到 actor、capability、resource scope、provider profile 或 vault boundary。 用户可以在设置中随时查看和撤销。

默认模式

默认是每次询问。高级用户可以切换到记住限定授权，或显式开启 YOLO 全访问。
